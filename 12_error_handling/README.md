## liblogger - простая библиотека журналирования C

Библиотека поддерживает 4 уровня журналирования:
- ERROR = 3,
- WARNING = 4,
- INFO = 6,
- DEBUG = 7.

Уровень по умолчанию ERROR. Библиотека выводит сообщения установленного уровня и всех более низких уровней.

 ### Пример использования liblogger:

```c++
#include "logger.h"
#include <_stdlib.h>

int main(void) {
    logger_set_env_prefix("MYAPP_");
    setenv("MYAPP_LOGGING_ENABLED", "1", 1);
    logger_init();
    logger_set_level(LOG_INFO);

    log_info("Связь со спутниками установлена");
    logger_shutdown();
    return 0;
}
```
### API
API очень простой и содержит несколько функций.

```c++
void logger_set_env_prefix(const char *prefix);
```
Префикс для переменных окружения, в которых библиотека будет искать настройки. Например, "MYAPP_". И, тогда,
все переменные будут начинаться на MYAPP_, например, "MYAPP_LOGGING_ENABLED".

```c++
void logger_init(void);
```
Библиотека инициализируется и считывает переменные окружения.

```c++
void logger_set_level(int level);
```
Установка важности уровня журналирования. Для него и более низких уровней будут выводиться сообщения. Можно динамически
менять в коде.

```c++
void logger_shutdown(void);
```
Завершение работы, закрытие файлов.

Доступны 4 макроса для вызова из кода и записи лог сообщений: log_error(), log_warning(), log_info(), log_debug().

### Конфигурация
Библиотека считывает перечисленные ниже переменные окружения. Если с помощью `logger_set_env_prefix()` установлен
префикс, то он будет добавлен перед названием переменной.

| Имя переменной   | Что делает                                                                                                            |
|------------------|-----------------------------------------------------------------------------------------------------------------------|
| LOGGING_ENABLED  | Если установлено 1 или True, то библиотека начинает логирование. Если нет, то ни какие сообщения не будут выводиться. |
| LOG_LEVEL        | Уровень логирования: error, warning, info, debug или соответствующий номер.                                           |
| LOG_FILE         | Файл, в который будут писаться сообщения от всех уровней важности. Если не задан, то запись будет в stderr.           |
| ERROR_LOG_FILE   | Имя файла для сообщений важности ERROR. Остальные, если не указано, будут писать в LOG_FILE или stderr                |
| WARNING_LOG_FILE | Имя файла для сообщений важности WARNING. Остальные, если не указано, будут писать в LOG_FILE или stderr              |
| INFO_LOG_FILE    | Имя файла для сообщений важности INFO. Остальные, если не указано, будут писать в LOG_FILE или stderr                 |
| DEBUG_LOG_FILE   | Имя файла для сообщений важности DEBUG. Остальные, если не указано, будут писать в LOG_FILE или stderr                |

Если для определенного уровня важности определен отдельный файл, то запись в него будет осуществляться отдельным потоком.

Печать стека вызовов функций производится в уровне DEBUG.
